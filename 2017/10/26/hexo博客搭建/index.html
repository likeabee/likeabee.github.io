搭建博客基本方式：

  • WordPress
  • GitHubPages+Jekyll
  • GitHubPages+Hexo
    WordPress我也没有接触过，不过据说一般需要独立域名，并且对markdown不友好（这个比较致命）;Jekyll我之前也有用过，不过相比较而言，个人觉得稍微有点麻烦;然后就是我今天要讲的Hexo了，如果不绑定自己的域名，搭建在githubPages上是完全免费的，并且操作起来简直不能更easy！好了，正文开始：

    at first

首先，当然是先去github创建仓库啦（如果你没有github账号，请先注册~）

这里的仓库名就填your_username.github.io就好了

需要注意的时，这里的仓库名的写法是固定的，必须是username.github.io！（username就是前面那个Owner）比如我的就必须是likeabee.github.io，否则一会跑起来的时候可能会404…

then

然后，当然是装环境咯

  • git安装
    我的系统是mac OS，mac环境下是自带git的（只要你装了Xcode…）
    Windows的话请自行安装git
  • Nodejs安装
    不论是Windows还是macOS，都可以去官网下载Nodejs安装

  • hexo安装
    使用命令行npm工具安装（Node Package Manager）

    注意：由于npm的源在国外，受到天朝万里长城的影响，国内用户使用起来可能速度堪忧，建议使用国内的镜像资源，速度会有大幅提升（如果已经实现科学上网请自动忽略）

    先执行命令安装cnpm：
    $ npm install -g cnpm --registry=https://registry.npm.taobao.org
    然后使用cnpm安装（适用于其他用npm安装的操作）：
    $ cnpm install hexo-cli -g

    next

接下来，当然就是创建本地博客的目录啦

  • 本地目录创建
    cd到你想要放置博客的目录下，大概像这样：

    执行hexo初始化命令：
    hexo init username.github.io // username就是你建仓库时候的用户名

  • 安装主题（以next为例）
    hexo初始化完成后默认是自带一个landscape的主题的，当然有许多主题可供选择，可以根据自己的喜好来选择，这里以next为例
    cd进入刚刚创建的文件根目录…不用我说了吧…
    ……就是cd xxx/username.github.io //xxx内容我反正是不知道是什么

    注意：以后的一些操作命令也都要cd进入这个目录以后在执行

    克隆next主题
    $ git clone https://github.com/iissnan/hexo-theme-next themes/next
    …..
    命令执行完后博客文件夹根目录下的themes文件夹下应该多了一个next文件夹，在根目录和next文件夹下都有一个_config.yml配置文件，请打开这两个文件细细研究下（可以用文本编辑打开，建议使用sublime等编辑器打开，代码还是有高亮状态看着舒服…）一些基本的操作都可以通过修改里面的参数实现！里面的英文注释还是比较清晰的…（看不懂请努力学习英语：）
    替换theme
    theme: next

additional content

好了，那么博客基本已经搭建好了，接下来就要开始写文章咯
我的写作方式是markdown，你要用什么是你的事，我不管:)
这里我讲一下关于hexo的命令操作：
$ hexo new page xxx这条命令会在source文件夹（不是themes里面主题下的source）下生成一个名为xxx的文件夹，文件夹里有一个index.md…= =…怎么说呢，这不是你写的文章，而是点击菜单栏时出现的页面，比如说菜单栏里有个tags，执行$ hexo new page tags并部署以后，你点击tags（标签）按钮出现的就是刚刚生成./source/tags/index.md的内容了
$ hexo new xxx这条命令才是真正意义上的写文章时执行的命令，执行完后，在source/_posts文件夹下生成一个同名xxx.md文件。你可以在这个md文件中写文章咯！

其他的请自己百度，谢谢(不想写了)

很简单你只需要在github的仓库中建一个CNAME的文件，把你的域名输进去就好了
…什么？…不会?……
在source文件夹下（是哪个source自己想…）新建一个CANME文件，没有后缀！然后编辑器打开，输入你的域名，保存，然后重新部署一遍就好了！

其实在github上直接建一个CANME文件输入域名也是ok的，但是这里不推荐，因为一旦重新部署了hexo上传时会把github上的项目文件覆盖一遍（你本地并没有CANME文件）这样，CNAME就消失了…除非你熟悉git操作，先把线上的代码fetch下来再push你的修改…不说了…
